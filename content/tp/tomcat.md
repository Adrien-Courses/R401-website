> Tomcat is a Web Container. It supports APIs in the presentation layer described
earlier. In addition, it supports JDBC and JPA also. It is easy to use and configure,
and could be a good option if you do not want to use EJBs.

## Tomcat VS GlassFish
- Tomcat est un conteneur de servlets et un serveur HTTP léger + JPA
- GlassFish est un serveur d'applications Java EE complet.
  - GlassFish existe en deux versions : Web Profile et Full Platform. Web Profile est
comme Tomcat, elle n'inclut pas le support des EJB
